## Principles

These principles apply uniformly to both repositories (AstroStack and
[AstroUI](https://github.com/x42en/astro-stack-ui)). Examples below are
Python-flavoured.

1. **English only.** Code, comments, commit messages, pull requests,
   documentation and any user-visible string are written in English. No mixed
   languages.

2. **Documentation comments.** Every public function, method, and class carries
   a docstring suitable for autogenerated documentation (Google or reST style,
   pick one per module and stick to it).

   ```python
   def stack_lights(frames: list[Path], rejection: str = "winsorized") -> Path:
       """Stack the given light frames using Siril.

       Args:
           frames: Calibrated light frames to stack.
           rejection: Pixel-rejection algorithm; one of ``winsorized``,
               ``sigma`` or ``percentile``.

       Returns:
           Path to the produced master light.

       Raises:
           PipelineError: If Siril returns a non-zero exit code.
       """
   ```

3. **Strict typing.** Every function and method carries explicit type
   annotations. `mypy` (or the project's configured type-checker) must pass
   clean. Avoid `Any` — reach for `object`, `TypeVar`, `Protocol`, or
   `TypedDict` first; if `Any` is genuinely required, justify it inline.

4. **Variable typing.** Annotate locals when the type is not obvious from the
   right-hand side, especially across function boundaries.

5. **Single responsibility.** Each class does one thing. Prefer composition
   over deep inheritance. Mix-ins beat base classes when behaviour is
   additive.

6. **Reuse before duplication.** Shared types live in `app/domain/`, shared
   logic in `app/core/` or a focused module. Extract on the second occurrence,
   never wait for the third.

7. **File size.** Target **under 500 lines** per file; **hard ceiling at
   1 000**. Split a file before it grows past the target.

8. **Fail fast and loud.** Validate inputs at boundaries (HTTP handlers,
   service entry points, deserialisers) and raise typed exceptions with clear
   messages. Inside the perimeter, **do not** add defensive null checks for
   states that cannot occur — let the program crash so the bug surfaces.

9. **No backward compatibility during migrations.** A breaking change ships
   with the migration that performs it; the old code path is removed in the
   same commit, not deprecated for an unbounded period.

10. **No dead code.** Remove unused imports, functions, parameters, and
    branches in the same PR that orphans them. If you cannot delete it now,
    you should not have written it.

11. **No duplication.** If you find yourself copying logic, extract it.

12. **Tests for every public surface.** New routes ship with at least one
    integration test; new pure functions ship with unit tests. Bug fixes ship
    with the test that would have caught the bug.

13. **Clean commits.** One logical change per commit. Tests pass on every
    commit (no "fix tests" follow-ups within a PR).

14. **Quality gates green before pushing.** `ruff check`, `ruff format
    --check`, `mypy` (where configured) and `pytest` must pass locally before
    you push.

## Project layout

```
app/
  api/               FastAPI routers, middleware
    middleware/        request logging, CORS, auth
    v1/                versioned route modules (sessions, jobs, profiles, gallery, catalog, ws/)
  core/              cross-cutting infrastructure (config, database, errors, logging, base models)
  domain/            pure dataclasses / SQLModel entities (no I/O)
  infrastructure/    adapters to external systems (queue, repositories, storage)
  pipeline/          orchestrator + steps + retry policy + adapters to external tools
    steps/             one module per pipeline step
    adapters/          wrappers around Siril, ASTAP, GraXpert, Cosmic Clarity
  services/          application-layer services orchestrating domain + infra
  workers/           ARQ worker settings + tasks
alembic/             migration scripts
scripts/             auxiliary executables (Siril macros, model bootstrap)
tests/
  unit/              fast, no I/O
  integration/       database + queue + filesystem
```

Boundary rules:

- `domain/` may not import from `services/`, `infrastructure/`, `api/` or
  `pipeline/`. It is the dependency root.
- `services/` may import from `domain/` and `infrastructure/`. Not from
  `api/`.
- `api/` is the only place that knows about HTTP/WebSocket transport.
- `pipeline/` may use `domain/` and `infrastructure/`. It is consumed by
  `workers/`.

## Local setup

### Prerequisites

- Python 3.12+
- PostgreSQL 15+ (locally or via `docker compose up postgres`)
- Redis 7+ (locally or `docker compose up redis`)
- Siril, ASTAP, GraXpert and Cosmic Clarity binaries on `PATH` if you intend
  to run the pipeline outside of Docker (the Docker images bundle them).

### Bootstrap

```bash
pip install -e ".[dev]"
cp .env.example .env

export DATABASE_URL="postgresql+asyncpg://astro:astro@localhost:5432/astrostack"
export REDIS_URL="redis://localhost:6379/0"

alembic upgrade head
uvicorn app.main:app --reload --port 8080

# In another shell
python -m arq app.workers.settings.WorkerSettings
```

## Coding standards

### Style

- `ruff format` is the source of truth for formatting — do not argue with it.
- `ruff check` enforces lint rules. Fix warnings, do not silence them with
  `# noqa` unless the rule itself is wrong for the case at hand (rare).

### Imports

Group imports as: standard library → third-party → first-party
(`from app.…`). `ruff` enforces this automatically.

### Errors

Use the typed exceptions defined in `app/core/errors.py`. Never raise bare
`Exception`. HTTP handlers translate domain exceptions to status codes via the
configured exception handlers; do not raise `HTTPException` from inside
services.

### Async

Database, queue, and HTTP I/O is async. Synchronous code (`time.sleep`,
blocking HTTP libraries) must not appear inside `async def` functions — use
`asyncio.to_thread` or refactor.

## Testing

```bash
pytest                           # everything
pytest tests/unit                # fast suite
pytest -k profile_sharing        # focused
pytest -x --ff                   # stop on first failure, retry failures first
```

- Unit tests live in `tests/unit/` and must not touch the network, database,
  or filesystem outside of `tmp_path`.
- Integration tests live in `tests/integration/` and use the test database
  fixtures in `tests/conftest.py`.
- Mark slow GPU-only tests with `@pytest.mark.gpu` and skip them in CI by
  default.

## Database migrations

Schema changes live in `alembic/versions/`.

```bash
# Create a new migration
alembic revision --autogenerate -m "add_observation_alerts_table"

# Inspect the generated file, edit if necessary, then apply
alembic upgrade head

# Roll back one revision
alembic downgrade -1
```

Rules:

- One logical schema change per revision.
- Migrations are **forward only** in production. Down-revisions must exist for
  developer convenience but are not relied upon for rollback.
- A migration that breaks code is paired with the code change in the **same**
  PR — see principle 9.

## Adding a feature

Typical recipe for a new endpoint that exposes a new domain capability:

1. Model the data in `app/domain/` (SQLModel entity, Pydantic DTOs).
2. Generate and review an Alembic migration.
3. Implement the persistence layer in `app/infrastructure/repositories/`.
4. Implement the application logic in `app/services/`.
5. Expose the endpoint in `app/api/v1/`.
6. Wire the new router into `app/main.py` if it is a new module.
7. Add unit tests for the service and integration tests for the endpoint.
8. Update [README.md](./README.md) features table if user-visible.

Adding a pipeline step:

1. Drop a module in `app/pipeline/steps/`.
2. Subclass `BaseStep` (`app/pipeline/base_step.py`).
3. Register it in the orchestrator and in the relevant presets.
4. Wrap any external CLI in `app/pipeline/adapters/`.
5. Add a unit test using a stubbed adapter.

## Common pitfalls

- **Synchronous I/O in async code** — silently blocks the event loop. Always
  use the async client of a library, or `asyncio.to_thread` for legacy CPU-
  bound or blocking calls.
- **Forgotten Alembic head merge** — running two feature branches in parallel
  produces a multi-head state. Merge with `alembic merge heads -m "..."`.
- **Missing GPU in worker container** — confirm `nvidia-smi` works **inside**
  the worker container. The Cosmic Clarity steps fail silently if CUDA is not
  visible.
- **Inbox watcher** — the watcher waits `SESSION_STABILITY_DELAY` seconds of
  inactivity. During tests, use the synchronous ingestion helpers in
  `tests/conftest.py` instead of dropping files and waiting.
